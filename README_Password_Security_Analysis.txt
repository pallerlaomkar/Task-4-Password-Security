
PASSWORD SECURITY ANALYSIS PRACTICAL
===================================

Project Title:
Password Security Analysis Using Hashcat and John the Ripper

------------------------------------------------------------
1. Overview
------------------------------------------------------------
This project demonstrates how weak passwords can be compromised using
password cracking techniques and highlights the importance of strong
authentication mechanisms.

The practical covers:
- Password hashing concepts
- Hash identification
- Dictionary-based password cracking
- Analysis of weak passwords
- Defensive security measures

------------------------------------------------------------
2. Tools Used
------------------------------------------------------------
Primary Tools:
- Hashcat
- John the Ripper

Alternative Tools:
- Online Hash Identifiers

------------------------------------------------------------
3. Environment
------------------------------------------------------------
Operating System:
- Kali Linux

Wordlist Used:
- rockyou.txt

------------------------------------------------------------
4. Practical Steps Summary
------------------------------------------------------------
1. Understood password storage mechanisms (hashing vs encryption)
2. Identified hash types such as MD5 and SHA-1
3. Generated password hashes for weak sample passwords
4. Performed dictionary attacks using Hashcat
5. Verified results using John the Ripper
6. Compared dictionary and brute-force attacks
7. Analyzed reasons for weak password failures
8. Studied the importance of Multi-Factor Authentication (MFA)
9. Documented observations and security recommendations

------------------------------------------------------------
5. Observations
------------------------------------------------------------
- Weak passwords were cracked quickly
- MD5 and SHA-1 hashes are insecure
- Dictionary attacks are highly effective against common passwords
- Strong hashing algorithms resist cracking attempts

------------------------------------------------------------
6. Security Recommendations
------------------------------------------------------------
- Use long and complex passwords
- Avoid common and predictable passwords
- Use strong hashing algorithms like bcrypt or Argon2
- Enable Multi-Factor Authentication (MFA)
- Implement account lockout policies

------------------------------------------------------------
7. Conclusion
------------------------------------------------------------
This practical provided hands-on knowledge of password attacks and
defensive mechanisms. It emphasizes the need for strong authentication
practices to protect user accounts.

------------------------------------------------------------
8. Final Outcome
------------------------------------------------------------
Knowledge of password attacks, password cracking techniques, and
effective security defenses.
